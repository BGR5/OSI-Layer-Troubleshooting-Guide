Problem #10: Layer 2 - Switch Port Not Forwarding Frames

A switch port may fail to forward Ethernet frames due to misconfigurations or hardware issues at the Data Link Layer (Layer 2). Although physical connectivity might appear normal (link lights are on), devices connected through that port cannot communicate.

Common Causes of Switch Port Forwarding Issues

  • Port is administratively disabled (shutdown)
  • Port is in error-disabled state due to security violations or spanning tree problems
  • Incorrect VLAN assignment or mismatch
  • Faulty or loose cables or hardware failure
  • MAC address table corruption or flooding

How to Fix:

  1. Check Port Status
    Use the command show interfaces status on the switch to verify if the port is enabled and operational.
    Example output should show the port as connected and not disabled or err-disabled.
  2. Enable the Port If Disabled
    If the port is administratively down, enter interface configuration mode and use no shutdown.
    Switch# configure terminal
Switch(config)# interface GigabitEthernet0/1
Switch(config-if)# no shutdown
  3. Check VLAN Assignment
    Ensure the port is assigned to the correct VLAN with show vlan brief.
    Correct mismatches by reassigning the VLAN:
    Switch(config-if)# switchport access vlan 10
  4. Verify Spanning Tree Status
    Check if the port is blocked by Spanning Tree Protocol (STP) using show spanning-tree interface GigabitEthernet0/1.
    If the port is in blocking or err-disabled state due to STP, investigate root cause (loops, misconfigurations) and resolve.
    You may re-enable err-disabled ports:
    Switch(config)# errdisable recovery cause stp
Switch(config)# errdisable recovery interval 300
  5. Inspect Physical Connectivity
    Check cables, connectors, and SFP modules. Replace faulty hardware.
    Try connecting a known-good device or cable to isolate the issue.
  6. Clear MAC Address Table
    If MAC address table corruption or flooding is suspected, clear it with:
    Switch# clear mac address-table dynamic
    This forces the switch to relearn MAC addresses.
  7. Test Connectivity
    After fixes, test connectivity by pinging devices on the same VLAN or using show mac address-table to verify addresses are learned.
