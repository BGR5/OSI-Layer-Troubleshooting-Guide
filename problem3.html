Problem #3: Port Blocking Issue

A client device (e.g., PC) cannot connect to a server using a specific TCP or UDP port (e.g., port 80 for HTTP, port 443 for HTTPS, or port 25 for SMTP). The connection times out or fails even though IP connectivity (ping) might be working.

Common Causes of Port Blocking Issue

  • Firewall Rules (Local or Network)
  • NAT (Network Address Translation) Issues
  • Misconfigured Router or ACLs (Access Control List)

How to Fix:

  1. Misconfigured Router or ACLs (Access Control List)
    Access Control Lists (ACLs) are used in routers and switches to control which packets are allowed or denied based on criteria such as IP addresses, protocols, and ports. If a port (like TCP port 80) is being blocked, an ACL might be explicitly denying it. This can happen unintentionally due to a misconfigured rule or incorrect placement of the ACL.
  2. Basic IP Connectivity
    Command: ping [destination IP]
    Purpose: Ensure Layer 3 (Network Layer) connectivity is working.
  3. Check for ACL (Access Control List) Rules
    If application access fails, check whether ACLs on the router are blocking the specific port. Use the show access-lists command to display active ACL rules. Look for any lines denying traffic based on TCP or UDP port numbers, e.g., deny tcp any any eq 80 blocks HTTP traffic.
    If such a rule exists and is applied on the correct interface and direction (e.g., ip access-group 100 in), it prevents access. You can modify or remove the rule depending on your needs.

Then retest by opening Desktop → Web Browser and typing the server’s IP (e.g., 192.168.2.10). The page should load correctly if port 80 is no longer blocked.
